# Incidente SharePoint 401 - 2026-03-05

## Resumen

Se detectaron errores `401` y `non-2xx` al leer/archivar PDFs de presupuestos y facturas en `sharepoint-storage`.

Escenarios observados:

1. Lectura de PDFs archivados desde `ArchivedPdfViewer`.
2. Archivado automatico tras `SENT` (presupuesto) o `ISSUED` (factura).

## Causa principal

Habia validacion doble de autenticacion:

- Gateway de Edge Function (`verify_jwt=true`).
- Validacion interna en `getAuthorizedUser()` (token + usuario autorizado en `internal.authorized_users`).

En ciertos flujos, el gateway devolvia `401` antes de ejecutar la logica interna y sin permitir fallbacks de recuperacion.

## Cambios aplicados

### Frontend

- `src/pages/nexo_av/shared/lib/supabaseSession.ts`
  - Nuevo helper `forceRefreshAccessToken()`.
- `src/pages/nexo_av/shared/lib/salesDocumentArchive.tsx`
  - Reintento automatico con refresh cuando `sharepoint-storage` devuelve `401/403`.
  - Mensajes de error mas especificos para auth.
- `src/pages/nexo_av/shared/components/ArchivedPdfViewer.tsx`
  - Descarga por `fetch` directo a la Edge Function para capturar `status` real.
  - Reintento con refresh de token.
  - Fallback por metadata canonica (`get-sales-metadata`) cuando el `filePath` persistido no coincide.
- Paridad responsive (desktop/mobile) en detalle de documentos:
  - `src/pages/nexo_av/desktop/pages/QuoteDetailPage.tsx`
  - `src/pages/nexo_av/mobile/pages/MobileQuoteDetailPage.tsx`
  - `src/pages/nexo_av/desktop/pages/InvoiceDetailPage.tsx`
  - `src/pages/nexo_av/mobile/pages/MobileInvoiceDetailPage.tsx`

### Edge Function

- `supabase/functions/sharepoint-storage/index.ts`
  - `getAuthorizedUser()` con fallback por email activo en `internal.authorized_users`.
  - Relink de `auth_user_id` cuando hay desalineacion.

## Despliegue ejecutado

- Funcion: `sharepoint-storage`
- Version activa: `9`
- Despliegue con `--no-verify-jwt` para evitar bloqueo de gateway y mantener la autenticacion en la validacion interna de la funcion.

## Verificaciones realizadas

- MCP SQL: usuarios autorizados activos con `auth_user_id` valido.
- MCP SQL: documentos emitidos/enviados del dia sin faltas de metadata (`archived_pdf_path` y `archived_pdf_file_name`).
- MCP Functions: `sharepoint-storage` activa en version `9`.

## Nota operativa

Los PDFs subidos manualmente a SharePoint por script pueden existir fisicamente y no ser legibles desde ERP si no tienen metadata canonica persistida en DB (`archived_pdf_path`, `archived_pdf_file_name`, `sharepoint_item_id`) para ese documento ERP.
